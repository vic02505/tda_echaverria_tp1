\section{Ejercio \#1: Comparación de Monedas}

\subsection{Problema a resolver}

Se tiene una bolsa con $n$ monedas de igual denominación, de las cuales exactamente una es falsa y es más liviana que el resto. 
El objetivo es identificar la moneda falsa sin ordenar por peso y usando División y Conquista.


\subsection{Algoritmo de División y Conquista}

La entrada del algoritmo es una lista de monedas representadas por su peso. Se asume que todos los pesos son iguales, a
excepción del peso de la moneda falsa.

El algoritmo divide la lista en dos. En cada paso compara las monedas de las posiciones centrales:

\begin{enumerate}
	\item Si las dos monedas centrales son distintas, la más liviana es la falsa.
	\item Si son iguales, se llama recursivamente a las dos partes y se repite el algoritmo.
\end{enumerate}

La cantidad de pesadas se contabiliza con cada comparación entre dos monedas, representando la cantidad de veces que se pesan
dos monedas hasta conseguir la falsa.

\subsection{Complejidad temporal y cantidad de pesadas}

Dado que el algoritmo se trata de un algoritmo de Divide y Conquista, la complejidad viene determinada por la ecuación de 
recurrencia del mismo:

$T(n) = A.T(n/B) + O(n^{C})$

Donde, $A = 2$ debido a que se hacen dos llamadas recursivas en cada paso, $B = 2$ porque en cada paso el algoritmo parte el
problema en dos, y $C = 0$ porque los costos adicionales incurridos en cada paso son $O(1)$.

La ecuación de recurrencia resultante es:

$T(n) = 2T(n/2) + O(1)$

Por el teorema maestro, la complejidad del algoritmo es: $O(n)$

\subsection{Metodología adoptada}

Se evaluó el algoritmo con sets de datos de distintos tamaños $n = \{100, 500,\allowbreak 1000,\allowbreak 5000,\allowbreak 10000,\allowbreak 50000\}$. Para cada tamaño se generó un 
archivo con una única moneda falsa y se ejecutó:

\begin{verbatim}
python3 ejercicios/ej-dyc/monedas.py <tamaño_set_de_datos>
\end{verbatim}

El tiempo se midió con la función time del sistema y se registró la cantidad de pesadas. Los resultados se 
guardaron en una tabla y se graficaron junto con una curva teórica lineal $O(n)$.

\subsection{Relación entre tiempo de ejecución y pesadas}

Es razonable vincular el tiempo de ejecución del algoritmo con la cantidad de pesadas debido a que cada paso del algoritmo implica realizar una 
cantidad constante de pesadas, y  el algoritmo tiene un crecimiento lineal, por lo tanto la cantidad de pasadas crece de forma proporcional al algoritmo,
es decir, linealmente ($O(n)$).
