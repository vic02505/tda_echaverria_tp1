\section{Ejercicio \#1: Comparación de Monedas}

\subsection{Problema a resolver}

Se tiene una bolsa con $n$ monedas de igual denominación, de las cuales exactamente una es falsa y es más liviana que el resto. 
El objetivo es identificar la moneda falsa sin ordenar por peso y haciendo uso de División y Conquista.

\subsection{Algoritmo de División y Conquista}

La entrada del algoritmo es una lista de monedas representadas por su peso. Se asume que todos los pesos son iguales, a
excepción del peso de la moneda falsa, es decir: $[a, b, b, b, b, b, b]$

En cada paso el algoritmo parte entrada en dos (a la lista) y calcula el promedio de cada parte.

\begin{enumerate}
	\item Si el promedio izquierdo es menor al promedio derecho se vuelve a llamar al algoritmo pero usando a la parte
	izquierda lista.
	\item Si el promedio derecho es menor al promedio izquierdo se vuelve a llamar al algoritmo pero usando a la parte
	derecha lista
\end{enumerate}

- Casos base:

1. Si la lista es de longitud uno, se retorna como moneda a su único elemento.

2. Si la lista es de longitud dos se debe devolver la moneda de menor pesos entre los dos elementos.

- Pesadas en el algoritmo:

Las pesadas son las veces que se pesan monedas para encontrar a la moneda falsa. En el algoritmo se pesan las monedas cada
vez que se calculan los promedios de cada una de las partes, y cuando la longitud de la lista de monedas es uno o dos.

\subsection{Complejidad temporal y cantidad de pesadas}

Dado que el algoritmo se trata de un algoritmo de Divide y Conquista, la complejidad viene determinada por la ecuación de 
recurrencia del mismo:

$T(n) = A.T(n/B) + O(n^{C})$

Donde, $A = 1$ debido a que se hacen dos llamadas recursivas en cada paso, $B = 2$ porque en cada paso el algoritmo parte el
problema en dos, y $C = 1$ porque los costos adicionales incurridos en cada paso vienen del calculo del promedio, el cual
es $O(n)$.

La ecuación de recurrencia resultante es:

$T(n) = 1T(n/2) + O(n)$

Por el teorema maestro, la complejidad del algoritmo es: $O(n)$

\subsection{Metodología adoptada}

Se evaluó el algoritmo con sets de datos de distintos tamaños $n = \{100, 500,\allowbreak 1000,\allowbreak 5000,\allowbreak 10000,\allowbreak 50000\}$. Para cada tamaño se generó un 
archivo con una única moneda falsa y se ejecutó:

\begin{verbatim}
python3 ejercicios/ej-dyc/monedas.py <tamaño_set_de_datos>
\end{verbatim}

El tiempo se midió con la función time del sistema y se registró la cantidad de pesadas. Los resultados se 
guardaron en una tabla y se graficaron junto con una curva teórica lineal $O(n)$.

\subsection{Relación entre tiempo de ejecución y pesadas}

Es razonable vincular el tiempo de ejecución del algoritmo con la cantidad de pesadas debido a que cada paso del algoritmo implica realizar una 
cantidad constante de pesadas, y  el algoritmo tiene un crecimiento lineal, por lo tanto, la cantidad de pesadas crece de forma proporcional al algoritmo,
es decir, linealmente ($O(n)$).
